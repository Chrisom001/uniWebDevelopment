Week 10 - XML

When looking at adding an RSS Feed to the site, I found during research that creating one is a fairly simple task as it’s simply an XML document and it can be updated manually with ease, although for sites which have grown in size it is generally recommended to use an automated creator if there are a lot of new articles in a short time. Another major factor when considering the implementation of RSS is that in recent years it’s use has been declining as only Internet Explorer remains the only browser to support it naturally as Mozilla removed native support from Firefox earlier this year.[1]

One advantage of an RSS feed is that it can make it much easier for users to be able to see what articles are available without necessarily having to visit the site which could improve traffic as well as it being in an easy to read format so they can browse it faster.

The alternative to RSS is called Atom which is newer and was designed to deal with some of the shortcomings of RSS. It has grown and has been adopted by a few companies as their preferred method but in others they do still use RSS or at times, the term RSS has become synonymous with both of them together.

On the newsletter website, RSS 2.0 has been used in order to provide a list of the articles with as much relevant information as possible within it. The XML file was created using notepad++ which made it possible to ensure that each of the tags was properly in place. Due to the simplicity of the XML format, updating the list shouldn’t be difficult.

Finally displaying those results was more difficult as it required the use of simpleXML in order to read the XML file into the PHP document. In order to get access to each of the records they each had to be parsed into a string before being displayed.
