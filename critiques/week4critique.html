Week 4 - Login/Register Security

The first feature of the login/register system which adds in an element of security is that data when the user is registering any empty space at the beginning or end of the input string is trimmed. More importantly however it is run through a strip_tags which removes a large majority of HTML, PHP and XML tags from the input string. This means that it’ll be harder for users to input code which could cause harm to the program itself or the database, as well as if the data is utilised after being taken from the database.

Another important security feature is prepared statements have been used within any queries which involve using any user input with the database. This is because if they were able to add anything extra to a query such as “105 OR 1=1” which would always be true. This is called SQL Injection and is one of the most common attack variants. The prepared statements protect against this as the values are put into the query after it’s “prepared” so it wouldn’t read the code in its entirety. Another part of this feature was strip slashes and html special character removal. These are important as the html special character feature allows nonstandard characters to be put into the database in an ASCII format rather than standard. This means that if they were to be used to try and attack the database, they’d be in a form which wouldn’t cause a problem as when the SQL was read, it would see “/” as “/” Finally when the input is put through the Strip slashes option, it looks for any forward or backslashes and removes these from the strings. This means that they can’t be used as part of an attack as well.

Additionally, when registering and logging in the password the user enters is run through a SHA256 hash. This is to prevent it from being easily read if the database were to be breached and the usernames and passwords were stolen. This is important as with the SALT that is applied to them, if a user reuses passwords then none of their other accounts are immediately compromised as it will take a hacker a significant amount of time to break the encryption.

The final security feature is when a user fails to logon correctly, instead of telling them what is wrong it simply tells the user than the username or password is incorrect. This is important as if a hacker knew if they had one of the options correct, then it would allow them to focus on the other making their attempt to break into the system much more likely to succeed.
