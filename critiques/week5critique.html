Week 5 - Article Input Security

When it comes to the user inputting their articles into the database and finally onto the site there are several security precautions which have been put into place to protect these.

Like the login system the first level of protection comes from sanitising the inputs given by the user. This is done by running through all their input through a number of filters to try and protect the site and the database from harm by any user who are unscrupulous. The main checks done are to trim the input, remove any special HTML characters, strip the slashes from the input and finally strip the tags from the input.

The second level of security is to prevent just registered users from being able to add new articles to the site. This is done by each user being set at a basic level which when logging in, is set as part of the session. The site then looks at the data stored in the session and if they do have a contributor level set, it will show the “Add an article” option in the navbar. This is done through a simple if/else statement which has been tested with users of both authentication levels.

Another security feature is during the image uploading. In order to make sure that the user isn’t inputting files which aren’t what they seem, the program checks all files that are being uploaded and to whether they are an image or not. If not, it will fail the upload and the user will be prompted to contact an administrator for assistance, as while this could be a mistake, it is something that would need to be investigated in more detail.

The final check on images is to ensure that they aren’t greater than 0.5mb. This both ensures that images aren’t too big for the site and causing longer load times for users, especially those on slower connections but to stop people inputting images which could be loaded with viruses or that are other files masquerading as an image. If an image that is too large is uploaded, it will also fail and tell the user to contact an administrator to look into this form them.
