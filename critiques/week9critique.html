Week 9 - Internet of Things (Week 3)

When looking at the best ways to be able to display the collected data on screen for users to be able to view, especially using JQuery mobile it was decided that using a table would be the best way to do this. This is because it becomes easier to show complex data in a clear format, with the last ten results being used to populate this.

This was done by installing the Jquery mobile pre-requisites and looking at the examples they had available on their site for tables. As part of this the best one for mobiles was one where a user could filter what columns they wanted to view. Due to the number of variables covered in the database, this seemed perfect for this part of the project. This was because as this was targeted towards the mobile aspect, the ability to hide or show columns would allow people with limited space, to be able to see only what they actually needed, rather than be forced to view every piece of data.

In order to get this working, the initial table elements were created and put into a variable to make it easier to display this later on. The data from the database was inserted into a variable, and then this was broken down further to get access to the json information stored within the sensorJson field within that. This data was then used within a for loop to create each individual line of the table. While this choice of implementation can take up more space than collapsible tabs, or menus, it is much clearer with data being more readily available to provide a better user experience.
